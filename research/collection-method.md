# Research Collection Method

## Purpose

This file documents how research material will be collected and organized for the project.

The assignment requires using Claude Code or Codex to support content collection through APIs or technical workflows, while also allowing manual collection for LinkedIn posts.

## Topic

AI-powered SEO content production for B2B SaaS.

## Tool Usage Plan

I will use Claude Code / Codex inside Cursor to support the technical and organizational parts of the research workflow.

The tools will be used for:

* Creating a clean repository structure
* Generating markdown templates for research notes
* Planning a YouTube transcript collection workflow
* Organizing collected content into consistent files
* Summarizing transcript material into structured research notes
* Documenting technical limitations and issues

## YouTube Collection Plan

For YouTube sources, I will use or attempt transcript/API-based collection where available.

Potential collection methods:

* YouTube transcript API or transcript extraction tools
* Supadata or similar transcript APIs where accessible
* Manual fallback only if transcript/API access is unavailable

For each YouTube source, I will collect:

* Expert name
* Video title
* Video URL
* Channel name
* Published date, if available
* Transcript availability
* Key takeaways
* Relevance to B2B SaaS AI SEO content production

I will not republish full transcripts unnecessarily. Instead, I will use transcripts as source material and save structured notes, summaries, and key takeaways.

## LinkedIn Collection Plan

LinkedIn content access is limited through public APIs, and the assignment allows manual collection of LinkedIn posts.

For LinkedIn sources, I will manually collect:

* Expert name
* LinkedIn profile URL
* Post URL
* Date, if visible
* Topic
* Summary
* Tactical takeaway
* B2B SaaS relevance
* AI SEO/content relevance
* Risk or limitation
* How it supports a future playbook

## Blog, Newsletter, Podcast, and Webinar Sources

For other public materials, I will collect:

* Source title
* Author/expert
* URL
* Date, if available
* Format
* Main idea
* Key takeaways
* Usefulness for future playbook development

## Research Quality Control

To avoid generic research, each source will be reviewed using the following questions:

1. Is this expert a real practitioner?
2. Is the source relevant to B2B SaaS?
3. Does it explain AI search, SEO, content production, audience research, or distribution?
4. Can the idea become part of a practical playbook?
5. Are there any risks, limitations, or conditions where the advice may not apply?

The research method has been defined. The next step is to begin collecting expert-specific content into the relevant folders.
